# Детектор скриншотов PWA

> Прототип для исследования методов обнаружения скриншотов в веб-приложениях

[![GitLab Pages](https://img.shields.io/badge/GitLab-Pages-orange?logo=gitlab)](https://gitlab.com)

---

## О проекте

Этот проект — исследовательский прототип, который демонстрирует различные техники обнаружения скриншотов в веб-приложении (PWA). Задача: изучить, насколько браузерные API позволяют детектировать момент, когда пользователь делает снимок экрана.

**Важно:** ни один из представленных методов не даёт 100% гарантии. Браузеры намеренно ограничивают доступ к системным событиям в целях безопасности и конфиденциальности. Проект носит **исследовательский и образовательный характер**.

### Возможные применения

- Исследование приватности данных в веб-приложениях
- Разработка механизмов предупреждения о скриншотах (например, в финансовых/медицинских системах)
- Тестирование браузерных API и их поведения при системных событиях

## Методы обнаружения

| # | Метод | Принцип работы | Надёжность | Платформа |
|---|-------|---------------|------------|-----------|
| 1 | **FPS Drop** | Снижение частоты кадров ниже 65% от baseline во время снятия скриншота | Средняя | Все |
| 2 | **Clipboard API** | Проверка буфера обмена на наличие изображения | Средняя | Chrome/FF (HTTPS) |
| 3 | **Long Tasks** | Длинные задачи CPU (>200 мс) через PerformanceObserver | Низкая | Chromium |
| 4 | **Visibility API** | Страница переходит в скрытое состояние | Низкая | Все |
| 5 | **Keyboard Shortcuts** | Перехват PrintScreen, Cmd+Shift+3/4/5, Win+PrtSc | Высокая | Desktop |
| 6 | **Print Events** | События `beforeprint` / `afterprint` и media query `print` | Средняя | Desktop |
| 7 | **DPR Change** | Изменение `devicePixelRatio` инструментом захвата | Средняя | Все |
| 8 | **Focus/Blur Timing** | Быстрое переключение фокуса (<3 сек) | Низкая | Все |
| 9 | **Screen Capture API** | Обнаружение активного захвата через `getDisplayMedia` | Средняя | Chrome/FF |
| 10 | **WebGL Context Lost** | Потеря WebGL-контекста при GPU-операциях | Низкая | Все |

## Ограничения

- **Clipboard API** работает только по HTTPS и требует явного разрешения пользователя
- **Keyboard shortcuts** не перехватываются на системном уровне — браузер может не получить событие (особенно на macOS)
- **FPS Drop** — сильно зависит от устройства; на мощных машинах просадка может быть незаметна
- **Long Tasks** доступны только в Chromium-браузерах
- **Screen Capture** требует явного согласия пользователя на захват
- **WebGL Context Lost** крайне редко срабатывает в реальных сценариях

## Быстрый старт

Приложение — это единственный HTML-файл без зависимостей.

### Локально

```bash
git clone <repo-url>
cd pg-screenshot-detect
# Открыть в браузере:
open index.html
# или
python3 -m http.server 8080  # затем перейти на http://localhost:8080
```

> **Для Clipboard API** нужен HTTPS. Используйте `localhost` (он считается безопасным контекстом в большинстве браузеров) или настройте локальный HTTPS.

### Использование

1. Открыть `index.html` в браузере (Chrome или Firefox рекомендуется)
2. Нажать кнопку **«Запустить мониторинг»**
3. Подождать ~5 секунд (за это время собирается FPS baseline)
4. Сделать скриншот любым доступным способом
5. Наблюдать за карточками методов и логом событий

## Деплой на GitLab Pages

### Автоматический деплой (через CI/CD)

В репозитории уже есть файл `.gitlab-ci.yml`, который автоматически публикует приложение на GitLab Pages при пуше в ветку `main`.

#### Шаги:

1. **Создать репозиторий на GitLab** (если ещё не создан)

2. **Запушить код в ветку `main`:**
   ```bash
   git remote add origin https://gitlab.com/<username>/<project>.git
   git checkout -b main
   git add .
   git commit -m "Initial commit"
   git push -u origin main
   ```

3. **Включить GitLab Pages** (если требуется вручную):
   - Перейти в `Settings → Pages`
   - Убедиться, что Pages включены для проекта

4. **Дождаться выполнения CI/CD пайплайна:**
   - Перейти в `CI/CD → Pipelines`
   - Дождаться успешного выполнения job `pages`

5. **Открыть приложение** по адресу:
   ```
   https://<username>.gitlab.io/<project>/
   ```

#### Структура `.gitlab-ci.yml`:

```yaml
image: alpine:latest

stages:
  - deploy

pages:
  stage: deploy
  script:
    - mkdir -p public
    - cp index.html public/index.html
  artifacts:
    paths:
      - public
  rules:
    - if: $CI_COMMIT_BRANCH == $CI_DEFAULT_BRANCH
```

> **Примечание:** Pages публикуются только из дефолтной ветки (обычно `main`). Для тестирования на других ветках добавьте дополнительное правило в `rules`.

### Ручная публикация (без CI)

Если CI/CD недоступен, можно создать папку `public/` вручную и загрузить её содержимое:

```bash
mkdir public
cp index.html public/index.html
# Загрузить папку public/ на любой статический хостинг
```

### Другие варианты хостинга

| Платформа | Команда |
|-----------|---------|
| **GitHub Pages** | Включить в `Settings → Pages`, выбрать ветку |
| **Netlify** | `netlify deploy --dir . --prod` |
| **Vercel** | `vercel --prod` |
| **Surge.sh** | `surge . <your-domain>.surge.sh` |

> Для работы **Clipboard API** приложение **должно** быть размещено по HTTPS. Все перечисленные платформы предоставляют HTTPS автоматически.
